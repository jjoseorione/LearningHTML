Entrada de sólo lectura

Descripción

Otra propiedad que podemos asignarle a las entradas de un formulario es el elemento readonly. Esta propiedad es sólo útil para entradas tecleadas (texto, textarea, número, correo, color, etc.).

Si definimos la propiedad readonly a un conrol, el mismo es de sólo lectura y no podemos modificar su contenido. Cuando una entrada tiene la propiedad readonly, el input toma foco, pero no se puede cambiar su contenido. La diferencia con la propiedad disabled es que con ésta el input no toma foco y generalmente aparece con un color que indica que el input está deshabilitado.

Ejemplo

Se confeccionará una página que muestre los términos y condiciones de una aplicación o página. Ésta se debe de encontrar en un textarea con la propiedad disabled, para que de este modo pueda ser leído o copiado, pero no borrado. El código sería el siguiente:

<form>
<fieldset>
<legend>Instalación</legend>
<textarea rows="10" cols="55" readonly>Términos y condiciones: Al instalar nuestro software usted acepta el uso de sus datos estadísticos para realización de pruebas de rendimiento y mejoramiento de nuestro software. En caso de un error, su sistema le preguntará si desea enviar un informe de errores. Al presionar "sí" usted acepta mandar una instantánea del estado de su máquina al momento del error. Copie y pegue este aviso en un bloc de notas para tenerlo disponible.
</textarea>
<br><input type="radio" name="acepta" value="true">Acepto los términos y condiciones.
<br><input type="radio" name="acepta" value="false">No acepto los términos y condiciones.
<br><br>
<button type="submit">Cancelar</button> <button type="submit">Siguiente</button>
</fieldset>
</form>

Y éste sería el resultado:

Instalación
Acepto los términos y condiciones.
No acepto los términos y condiciones.
